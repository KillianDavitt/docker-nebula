Intro

This page described how to setup and configure a development workflow for building Yesod based applications (but also any other haskell development) using the Stack build system, with all compilation and execution running on a Docker instance that isolates the compilation/execution of your system from the peculiarities of your host operating system dependencies.

  • Stack is a build system for haskell. It takes care of installing necessary haskell backends for you, installing all packages your project needs dynamically (say goodbye to dependency hell), it builds, tests and benchmarks your project.Think of it as a replacement to cabal and more.

  • Docker is a software containerisation platform allowing you to pick stable development environments to host your toolsets. What sounds like an unnecessary layer in fact greatly simplifies software development. why docker blog post. We will use Docker to host our Haskell build.
